# Cisco Router Wizard

Простий інструмент із GUI (Eel) для генерації конфігурацій Cisco-маршрутизаторів на основі введених даних.

## Зміст репозиторію

- `backend/` — Бекенд: генерація конфігів, валідація.
- `docs/` — Документація (Архітектура, Жива документація, Генерація).
- `scripts/` — Допоміжні скрипти (Лінтер документації).
- `tests/` — Юніт, інтеграційні та перформанс тести.

## Ключові документи

- 🏛️ **Архітектура**: [ARCHITECTURE.md](docs/ARCHITECTURE.md) — опис системних рішень та алгоритмів.
- 📖 **Жива документація**: [LIVING_DOCS.md](docs/LIVING_DOCS.md) — приклади використання через активні тести.
- ⚙️ **Технічна інструкція**: [generate_docs.md](docs/generate_docs.md) — як генерувати HTML-документацію.

## Швидкий старт (Windows)

```powershell
# Встановлення
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
py -3 -m pip install -r requirements.txt

# Запуск застосунку
py -3 -m backend.main
```

## Тестування та якість

### Запуск тестів
```powershell
py -m pytest tests/ -v
```

### Перевірка якості документації (Linter)
Проект використовує коментарі через `#` для опису функцій. Для перевірки покриття використовуйте:
```powershell
py scripts/check_docs.py
```

## Стандарти розробки

1. **Документування**: Кожна функція повинна мати технічний опис (коментарі `#`) над визначенням або в перших рядках тіла.
2. **Типізація**: Обов'язкове використання Type Hints.
3. **Тести**: Новий функціонал повинен покриватися тестами та додаватися до [LIVING_DOCS.md](docs/LIVING_DOCS.md).
