# Cisco Router Wizard

Простий інструмент із GUI (Eel) для генерації конфігурацій Cisco-маршрутизаторів на основі введених даних.

## Зміст репозиторію

- `backend/` — Бекенд: генерація конфігів, валідація.
- `docs/` — Документація (Архітектура, Жива документація, Генерація).
- `scripts/` — Допоміжні скрипти (Лінтер документації).
- `tests/` — Юніт, інтеграційні та перформанс тести.

## Ключові документи

- 🏛️ **Архітектура**: [ARCHITECTURE.md](docs/ARCHITECTURE.md) — опис системних рішень та алгоритмів.
- 📖 **Жива документація**: [LIVING_DOCS.md](docs/LIVING_DOCS.md) — приклади використання через активні тести.
- ⚙️ **Технічна інструкція**: [generate_docs.md](docs/generate_docs.md) — як генерувати HTML-документацію.

## Покрокова інструкція для розробника (Онбордінг)

Ця інструкція передбачає, що ви працюєте за комп'ютером із щойно встановленою операційною системою (Windows).

### 1. Необхідні залежності та програмне забезпечення
Перед початком роботи переконайтеся, що на вашому комп'ютері встановлені наступні програми:
- **Git**: Для клонування та роботи з репозиторієм ([завантажити](https://git-scm.com/downloads)).
- **Python**: Версія 3.10 або новіша ([завантажити](https://www.python.org/downloads/)). 
  *(Під час встановлення обов'язково поставте галочку **"Add python.exe to PATH"**!)*
- **Код-редактор**: Рекомендується **VS Code** або **PyCharm**.

### 2. Клонування репозиторію та налаштування середовища розробки
Відкрийте термінал (PowerShell) у папці, де ви хочете розмістити проект, та виконайте:

```powershell
# Клонування репозиторію
git clone <URL вашого репозиторію>
cd Cisco-Router-Wizard-CRW-

# Створення віртуального середовища Python
py -3 -m venv .venv

# Активація віртуального середовища
.\.venv\Scripts\Activate.ps1
```
> **Примітка**: Якщо у PowerShell виникає помилка виконання скриптів (`execution of scripts is disabled`), відкрийте термінал від імені адміністратора та виконайте: 
> `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` і повторіть активацію.

### 3. Встановлення та конфігурація залежностей
Після успішної активації віртуального середовища (у терміналі з'явиться префікс `(.venv)`) встановіть усі необхідні бібліотеки проекту:

```powershell
py -m pip install -U pip
py -m pip install -r requirements.txt
```

### 4. Створення та налаштування бази даних (якщо потрібно)
**База даних у цьому проекті не використовується**. Застосунок призначений для локальної генерації текстових конфігурацій (Cisco IOS) "на льоту" через GUI і не зберігає стан користувачів або конфігурацій у базі даних (БД). Тому налаштовувати або підключати БД не потрібно.

### 5. Запуск проекту у режимі розробки
Щоб запустити застосунок, використовуйте наступну команду з кореня проекту:

```powershell
py -m backend.main
```
Це запустить бекенд-сервер та автоматично відкриє графічний інтерфейс застосунку у окремому вікні. 

### 6. Базові команди та операції

- **Запуск усіх тестів (юніт, інтеграційні тощо):**
  Будь-який новий функціонал обов'язково треба покривати тестами.
  ```powershell
  py -m pytest tests/ -v
  ```
- **Перевірка якості документації (Лінтер):**
  Проект використовує спеціалізовані коментарі (`#`) для опису функцій в коді. Перевірити їх наявність можна лінтером:
  ```powershell
  py scripts/check_docs.py
  ```

## Стандарти розробки

1. **Документування**: Кожна функція повинна мати технічний опис (коментарі `#`) над визначенням або в перших рядках тіла.
2. **Типізація**: Обов'язкове використання Type Hints.
3. **Тести**: Новий функціонал повинен покриватися тестами та додаватися до [LIVING_DOCS.md](docs/LIVING_DOCS.md).
