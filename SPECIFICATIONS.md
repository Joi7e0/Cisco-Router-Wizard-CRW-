# Специфікації ключових функцій (для тестування та рефакторингу)

## 1. validate_inputs (validate.py)

**Мета**: Перевірити коректність усіх основних вхідних даних перед генерацією

**Вхід**  
- 6 обов'язкових полів: 3×IP + 3×маска  
- routing_protocol (str)  
- router_id (str, тільки для OSPF)

**Вихід**  
"" або повідомлення "❌ Error: ..."

**Критичні тестові сценарії**  
- Усі поля порожні  
- Некоректний IP (256.x.x.x, текст, IPv6)  
- Некоректна маска (/33, 300.0.0.0, "255.255")  
- OSPF + порожній router_id  
- Коректні дані + зайві пробіли на початку/кінці

## 2. _mask_to_wildcard (protocols.py)

**Вхід**: маска "255.255.255.252"  
**Вихід**: "0.0.0.3"

**Критичні випадки**  
- /24 → 0.0.0.255  
- /30 → 0.0.0.3  
- /16 → 0.0.255.255  
- Некоректна маска → має кидати помилку  
- Порожній рядок → помилка

## 3. generate_protocol_config (protocols.py)

**Ключові сценарії**  
- Кожен підтримуваний протокол з 1–3 мережами  
- Невідомий протокол → "! No routing protocol selected"  
- Порожній список мереж  
- OSPF без router-id (очікується, що валідація пройшла раніше)  
- Велика кількість мереж (50–100)

## 4. generate_interface_config (generate.py)

**Ключові сценарії**  
- 0 інтерфейсів → порожній список  
- 1–10 інтерфейсів  
- Частина інтерфейсів з no shutdown, частина без  
- Різна кількість інтерфейсів і мереж → помилка на вищому рівні  
- Дублі інтерфейсів (не обробляється тут)

## 5. generate_full_config (generate.py)

**Найважливіші комбінації для тестування**  
1. Мінімальна конфігурація (тільки hostname + 1 інтерфейс)  
2. Повна конфігурація (усі функції увімкнено)  
3. Тільки OSPF + DHCP  
4. Тільки Telephony без ephone-dn  
5. SSH без паролів  
6. DHCP з порожньою маскою/мережею  
7. 50+ інтерфейсів (performance check)
