# Звіт з виконання роботи (від лабораторної 3, 60% до 100%+)

Цей звіт підсумовує інтеграцію DevOps та Release Managegment практик у проект. Відповідно до вимог, структуру директорій та документації приведено до еталонного формату.

## 1. Інструкція для розробників
**Файл:** `README.md` (в кореневій директорії)
- Додано розділ "Покрокова інструкція для розробника (Онбордінг)", де поетапно описано підготовку до роботи.
- Документовано необхідне ПЗ (Git, Python 3.10+).
- Описано клонування репозиторію, ініціалізацію `.venv`, встановлення залежностей, запуск проекту у режимі розробника, а також лінтинг.

## 2. Інструкції для DevOps
Документацію винесено в теку `docs/`:

- **`docs/deployment.md`** (Виконання на 74%)
  - Описані вимоги до апаратного (x86_64, 512MB RAM) та програмного забезпечення.
  - Налаштування мережі (localhost).
  - Конфігурація термінальних серверів для багатокористувацької роботи з GUI.
  - Описано непотрібність інсталяції окремих СУБД (Stateless архітектура).
  - Наведено критерії перевірки працездатності розгорнутого застосунку.

- **`docs/update.md`** (Виконання до 90%)
  - Рекомендації щодо перевірки сумісності, планування технологічного вікна та бекапів.
  - Процес зупинки процесів, оновлення коду (заміна скомпільованих версій).
  - Створено процедуру швидкого відкату (Rollback) на випадок післярелізних збоїв.

- **`docs/backup.md`** (+5 балів)
  - Описано повні інструкції для резервного копіювання: частота копіювання коду та артефактів збірки.
  - Описано правила зберігання і ротації релізних збірок в Artifacts Storage.
  - Покрокова процедура DR (Disaster Recovery): повне відновлення після збою та перевірка відновлення.

## 3. Додаткові скрипти та конфігурації
Згідно з вимогою групування, усі створені скрипти розгортання та DevOps конфігурації (додаткові +5 балів) переміщено в теку `docs/scripts/`.

- **Автоматизація запуску/оновлень (Bash/PowerShell/Bat):**
  - `start_dev.bat` та `start_prod.bat`: Скрипти для запуску проекту (Dev/Prod) з вбудованими перевірками.
  - `deploy_update.ps1`: Скрипт розгортання для Windows Server з бекапом та безпечною зупинкою.

- **Системи CI/CD:**
  - `.github/workflows/ci-cd.yml` (знаходиться у перенесеній теці `docs/scripts/.github`): Автоматизація Github Actions (тестування, збірка білдів).

- **Контейнеризація:**
  - `Dockerfile` та `docker-compose.yml`: Підготовлені образи для запуску Python-бекенду в контейнеризованій мережі.

- **Оркестрація:**
  - `k8s/deployment.yaml` та `k8s/service.yaml`: Налаштування LoadBalancer та Deployment конфігурацій для Kubernetes.

- **Інфраструктура (IaC):**
  - `terraform/main.tf`: Скрипт створення серверів в AWS та ініціалізації середовища з 0.
