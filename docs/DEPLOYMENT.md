# Інструкція з розгортання у виробничому середовищі (Production Deployment)

Цей документ призначений для Release Engineers та DevOps фахівців, і містить вказівки щодо підготовки та розгортання застосунку "Cisco Router Wizard" у виробничому середовищі (Production). 

Завдяки специфіці роботи (GUI застосунок на основі бібліотеки Eel), "виробниче середовище" зазвичай означає використання збірки (скомпільованого файлу) на кінцевих робочих станціях інженерів або на спільному термінальному сервері.

## 1. Вимоги до апаратного забезпечення
Застосунок є легковаговим і не створює значного навантаження на систему.
- **Архитектура:** x86_64 / ARM64.
- **Мінімальні вимоги до CPU:** 1 ядро (1.0 GHz або вище).
- **Мінімальні вимоги до пам'яті (RAM):** 512 MB (рекомендується 1 GB для забезпечення комфортної роботи рушія WebView/Chromium).
- **Дисковий простір:** Близько 150-200 MB для розміщення скомпільованого `.exe` файлу або запуску з віртуального середовища.

## 2. Необхідне програмне забезпечення
Залежно від способу доставки застосунку:
- **Для Build/CI сервера (збірка релізу):**
  - Python 3.10+
  - Git
  - Модуль `PyInstaller` (встановлюється через `pip`)
- **Для Цільової машини (Target Machine / Terminal Server):**
  - Вже встановлений сучасний браузер (Google Chrome або Microsoft Edge) для відображення інтерфейсу (Eel використовує їх).
  - Якщо використовується готовий `CiscoRouterWizard.exe`, додатково встановлювати Python **не потрібно**. Підтримуються ОС: Windows 10/11, Windows Server 2016+.

## 3. Налаштування мережі
- **Вхідні/Вихідні порти:** "Cisco Router Wizard" — це ізольований локальний інструмент. Eel запускає бекенд-сервер на випадковому вільному порту `localhost` (127.0.0.1), який доступний тільки локально. Відкривати чи прокидати зовнішні порти на файрволі/маршрутизаторі **не потрібно**.
- **Доступ до Інтернету:** Не вимагається. Застосунок може працювати у повністю ізольованих (Air-gapped) середовищах після його встановлення.

## 4. Конфігурація серверів
Якщо застосунок розгортається на **Terminal Server** (наприклад, Windows Server RDS) для надання доступу кільком співробітникам:
- Розмістіть `.exe` файл у загальній директорії (наприклад, `C:\Program Files\CiscoRouterWizard\`).
- Додайте ярлик до спільного робочого столу або додайте директорію до змінної `PATH`.
- Eel автоматично ізолює процеси. Кожен інженер під час запуску ініціюватиме власний локальний сервер на унікальному порту у своїй сесії RDP, конфліктів портів не виникатиме.

## 5. Налаштування СУБД (Бази даних)
**База даних не використовується.**
Проект призначений для миттєвої генерації текстових конфігурацій (Cisco IOS) "на льоту". Жодні профілі клієнтів, паролі чи створені конфігурації не зберігаються у постійному сховищі.
Отже, встановлення, налаштування чи підтримка SQL/NoSQL баз даних **не потрібні**.

## 6. Розгортання коду (Збірка релізу)
Рекомендується постачати застосунок кінцевим користувачам у вигляді єдиного `.exe` файлу, аби уникнути проблем із залежностями. Інструкція для DevOps/CI пайплайну:

1. Отримайте свіжий код стабільного релізу:
   ```powershell
   git clone <URL_репозиторію>
   cd Cisco-Router-Wizard-CRW-
   git checkout main # або конкретний release tag, наприклад v1.0.0
   ```
2. Ініціалізуйте середовище та встановіть залежності:
   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   pip install pyinstaller # для створення збірки
   ```
3. Створіть Production-збірку (єдиний файл без консольного вікна):
   ```powershell
   python -m eel backend/main.py frontend --onefile --noconsole --name "CiscoRouterWizard"
   ```
4. **Артефакт:** Готовий файл `dist/CiscoRouterWizard.exe` є виробничою версією проекту. Його можна завантажити у корпоративне сховище артефактів або безпосередньо передати користувачам.

## 7. Перевірка працездатності (Health Check)
Щоб верифікувати успішність розгортання:
1. Запустіть створений файл `CiscoRouterWizard.exe` на цільовій машині (або `python -m backend.main` у разі запуску з вихідних кодів).
2. **Критерій успіху 1 (Запуск):** Повинно відкритися вікно браузера із графічним інтерфейсом додатку. Жодних помилок в консолі (якщо вона увімкнена) бути не повинно.
3. **Критерій успіху 2 (Функціональність):** Заповніть базову інформацію (наприклад, введіть "TestRouter" у полі Hostname), пропустіть решту кроків і натисніть кнопку "Generate Configurations". Якщо на екрані з'явився згенерований текст (IOS конфіг) без винятків чи "зависань", система працює коректно. Спробуйте також скопіювати згенерований текст (кнопка "Copy to clipboard"), щоб перевірити роботу системного буфера обміну.
