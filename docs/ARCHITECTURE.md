# Архітектура Cisco Router Wizard (CRW)

## 1. Загальний огляд
CRW — це десктопний застосунок для автоматизації створення конфігурацій Cisco IOS. Застосунок побудований на базі Python (бекенд) та HTML/JS (фронтенд) з використанням бібліотеки **Eel** для комунікації.

## 2. Компоненти та взаємодія

### Фронтенд (Web UI)
- **Технології**: HTML5, CSS3, Vanilla JavaScript.
- **Роль**: Збір параметрів від користувача (IP, інтерфейси, протоколи).
- **Зв'язок**: Викликає Python-функції через `eel.generate_config(...)` та `eel.validate_input(...)`.

### Міст (Eel Bridge)
- Виконує роль прошарку між JavaScript та Python.
- Забезпечує передачу об'єктів (JSON) між середовищами.

### Бекенд (Python Core)
Розділений на три основні модулі:
1.  **`main.py`**: Точка входу, запуск Eel-сервера та експорт функцій для фронтенду.
2.  **`validate.py`**: Бізнес-логіка перевірки введених даних (IP-адреси, Hostnames, діапазони DHCP).
3.  **`generate.py`**: Ядро генерації конфігурації. Використовує **Jinja2** для рендерингу шаблонів.

## 3. Процес генерації конфігурації
Генерація відбувається ієрархічно:
1.  `generate_full_config` — головний оркестратор.
2.  Він послідовно викликає спеціалізовані генератори:
    - `generate_base_config` (Hostname/Interfaces)
    - `generate_protocol_config` (Routing)
    - `generate_security_config` (SSH/VTY)
    - `generate_dhcp_config`
    - `generate_nat_config`
    - `generate_snmp_config`
3.  Результат зшивається в єдиний список команд Cisco IOS.

## 4. Алгоритмічні рішення

### Розрахунок масок (Wildcard Mask)
Для протоколів OSPF та EIGRP використовується Wildcard Mask (зворотна маска). 
- **Логіка**: `255 - octet` для кожного октету.
- **Модуль**: `backend/protocols.py`, функція `_mask_to_wildcard`.

### Валідація IP-мереж
Використовується бібліотека `ipaddress` для перевірки належності IP-адреси до вказаної мережі та перевірки накладання підмереж.

## 5. Обробка помилок
- Валідатори повертають рядок помилки з префіксом `❌ Error:`.
- Фронтенд відображає ці помилки біля відповідних полів вводу в реальному часі.

## 6. Стратегія тестування
- **Unit-тести**: Перевірка окремих регулярних виразів та логіки масок.
- **Integration-тести**: Перевірка повного циклу генерації від входу (dict) до виходу (список команд).
- **Performance-тести**: Замір швидкості рендерингу шаблонів під навантаженням.
